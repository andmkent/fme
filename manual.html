Fourier-Motzkin Elimination for Integer Systems

Andrew Kent <andmkent@indiana.edu>}

This package is a simple implementation of the Fourier-Motzkin elimination method. It reasons about systems of linear inequalties (SLIs) and currently has two primary functions:

  1. Decide the satisfiability of a SLI (over reals).

  2. Decide if one SLI implies another (over integers).

The linear inequalities are of the form (L1 ≤ L2), where L1 and L2 are simple linear combinations of the form ax + by + ... + c.

1 Some Quick Examples

Examples:

> (let ([assumptions (sli (leq (lexp (7 'x))
                               (lexp (29 #f))))]
        [goal (leq (lexp (1 'x)) (lexp (4 #f)))])
    (if (sli-proves-leq? assumptions goal)
        (printf "7x ≤ 29\nimplies\nx ≤ 4\n")
        (printf "7x ≤ 29 \ndoesn't imply\n x ≤ 4")))

7x ≤ 29

implies

x ≤ 4

> (let ([assumptions (sli (leq (lexp (7 'x))
                               (lexp (29 #f))))]
        [goal (leq (lexp (1 'x)) (lexp (3 #f)))])
    (if (sli-proves-leq? assumptions goal)
        (printf "7x ≤ 29 \nimplies\nx ≤ 3")
        (printf "7x ≤ 29 \ndoesn't imply\nx ≤ 3")))

7x ≤ 29

doesn't imply

x ≤ 3

> (let ([assumptions (sli (leq (lexp (1 'x) (1 'y))
                               (lexp (1 'z)))
                          (leq (lexp)
                               (lexp (1 'y)))
                          (leq (lexp)
                               (lexp (1 'x))))]
        [goals (sli (leq (lexp (1 'x))
                         (lexp (1 'z)))
                    (leq (lexp (1 'y))
                         (lexp (1 'z)))
                    (leq (lexp (0 #f))
                         (lexp (1 'z))))])
    (if (sli-proves-sli? assumptions goals)
        (printf "x + y ≤ z ∧ 0 ≤ y ∧ 0 ≤ x \nimplies\nx ≤ z ∧ y ≤ z")
        (printf "x + y ≤ z ∧ 0 ≤ y ∧ 0 ≤ x \ndoesn't imply\nx ≤ z ∧ y ≤ z ∧ 0 ≤ z")))

x + y ≤ z ∧ 0 ≤ y ∧ 0 ≤ x

implies

x ≤ z ∧ y ≤ z

2 Linear Expressions

syntax

(lexp (a x) ...)

 
  a : exact-integer?
  x : any/c
Returns a linear expression of the form ax+ ... where a is the scalar coefficient and x is the variable. For constants, use #f as the variable. (Currently we merely represent these with a hash mapping variables to coefficients)

procedure

(lexp? e) → boolean?

  e : any/c
Tests if argument is a linear expression.

procedure

(lexp-scalar e x default) → any/c

  e : lexp?
  x : any/c
  default : any/c
Returns the scalar coefficient for the variable x in e, or default if x is not present. #f is used as the variable for constants.

procedure

(lexp-vars e) → (listof any/c)

  e : lexp?
Returns a list of the variables in this linear expression (in no particular order). #f is included if the expression has a constant.

procedure

(lexp-has-var? e x) → boolean?

  e : lexp?
  x : any/c
Returns #t if x is in e, else #f.

procedure

(lexp-scale e a) → lexp?

  e : lexp?
  a : exact-integer
Multiplies all scalar coefficients in e by a.

procedure

(lexp-remove e x) → lexp?

  e : lexp?
  x : any/c
Removes the term with the variable x from the linear expression e (no-op if x not present).

procedure

(lexp-set e x a) → lexp?

  e : lexp?
  x : any/c
  a : exact-integer?
Changes the coefficient for x to a (adding x if it is not in the linear equation).

procedure

(lexp-size e) → exact-nonnegative-integer?

  e : lexp?
Returns the number of terms in the linear expression (i.e. number of unique variables +1 if a constant is present).

procedure

(lexp-empty? e) → boolean?

  e : lexp?
Reports if the linear expression is entirely empty (i.e. equal to 0).

procedure

(lexp-subtract e1 e2) → lexp?

  e1 : lexp?
  e2 : lexp?
Returns the result of e1 - e2. For example (shown below), (2x + 3y - 1) - (2x + 42z - 1) = 3y - 42z

Example:

> (lexp-subtract (lexp (2 'x) (3 'y) (-1 #f))
            (lexp (2 'x) (-1 #f) (42 'z)))

'#hash((y . 3) (z . -42))

Note that lexp’s use hash tables for their representation, so their values are printed in the regular hash form.

procedure

(lexp-add1 e) → lexp?

  e : lexp?
Adds 1 to the constant in e.

Example:

> (equal? (lexp-add1 (lexp (1 #f) (5 'x)))
          (lexp (2 #f) (5 'x)))

#t

3 Linear Inequalities

struct

(struct leq (lhs rhs)
    #:extra-constructor-name make-leq)
  lhs : lexp?
  rhs : lexp?
A structure representing a less-than-or-equal-to inequality of the form lhs ≤ rhs.

procedure

(leq-exps ineq) → 
lexp? lexp?
  ineq : leq?
Returns the lhs and rhs of the inequality.

procedure

(lexp-contains-var? ineq x) → boolean?

  ineq : leq?
  x : any/c
Returns #t if x is in the lhs or rhs of ineq, else #f.

procedure

(leq-negate ineq) → leq?

  ineq : leq?
Negates ineq based on *integer math*, e.g. not (x <= y) implies (y+1 <= x).

procedure

(leq-isolate-var ineq x) → leq?

  ineq : leq?
  x : any/c
Converts an inequality into either (ax <= by + cz + ...) or (by + cz + ... <= ax) such that a is a positive integer and x appears on at most one side of the inequality. If x is not in ineq, it is a no-op.

procedure

(leq-join ineq1 ineq2 x) → leq?

  ineq1 : leq?
  ineq2 : leq?
  x : any/c
Takes a pair of inequalities of the forms (a1x <= l1) and (l2 <= a2x) and returns a2l1 <= a1l2.

procedure

(leq-trivially-valid? ineq1 ineq2) → boolean

  ineq1 : leq?
  ineq2 : leq?
Checks if ineq1 <= ineq2 is trivially true (i.e. they must both be constants).

4 Systems of Linear Inequalities

procedure

(sli ineq ...) → (listof leq?)

  ineq : leq?
Constructor for a system of linear inequalities. Currently just wraps list.

procedure

(sli-vars sys) → (listof any/c)

  sys : (listof leq?)
Returns a list of all of the variables in the inequalities of this system, with duplicates removed.

procedure

(sli-partition sys x)

 → 
(listof leq?) (listof leq?) (listof leq?)
  sys : (listof leq?)
  x : any/c
3-way partitions the system of inequalities based on how leq-isolate-var would re-order the inequality in terms of x:

  1. Inequalities of the form (ax <= by + cz + ...)

  2. Inequalities of the form (by + cz + ... <= ax)

  3. Inequalities which do not contain x

procedure

(sli-elim-var sys x) → (listof leq?)

  sys : (listof leq?)
  x : any/c
Reduces the system of linear inequalities, performing the proper algebraic eliminations (see sli-partition and leq-join) such that the system no longer contains x but maintains the same set of satisfying solutions with respect to the remaining variables.

This elimination is sound for *real number* solutions. (Since we are mainly concerned with unsatisfiability, however, this is not a huge issue, since if a system is unsatisfiable for reals it is also unsatisfiable for integers, thus this is also a sound method for checking unsatisfiability of integer problems as well).

procedure

(sli-satisfiable? sys) → boolean?

  sys : (listof leq?)
Reports if a given system of linear inequalities is satisfiable (i.e. does a real number solution) exist.

procedure

(sli-proves-leq? sys ineq) → boolean?

  sys : (listof leq?)
  ineq : leq?
Reports if a given system of linear inequalities, sys, implies the constraint ineq (this is valid for the integer domain of inequalities only, since we utilize integer inequality negation).

procedure

(sli-proves-sli? sys1 sys2) → boolean?

  sys1 : (listof leq?)
  sys2 : (listof leq?)
Reports if sys1 implies all of the constraints contained in sys2 (again, this is valid for the integer domain of inequalities only, since we utilize integer inequality negation).
